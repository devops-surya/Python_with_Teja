5. Members and Methods

Members / Attributes

Members are data stored in the class or object. They are like variables inside OOP.

balanceaccount_numbercustomer_name

Methods

Methods are functions written inside a class. They describe what the object can do.

withdraw()deposit()transfer()

6. Instance Level Method Calling

Instance level means object level. An instance method works on one specific object and uses self.

class BankAccount:
    def withdraw(self, amount: float):
        pass

    def transfer(self, amount: float, account_number: str):
        pass

How to call instance methods:

user1_account = BankAccount()
user1_account.withdraw(500)
user1_account.transfer(500, "8787878787")

7. Class Level Method Calling

A class method belongs to the class. It can be called without creating an object.

class BankAccount:
    @classmethod
    def bank_name(cls) -> str:
        return "HDFC"

BankAccount.bank_name()

Output:

"HDFC"

8. Class Level Member

A class level member is stored on the class itself. It is useful for common data shared by all objects.

class NewBankAccount:
    bank_name: str = "HDFC"

NewBankAccount.bank_name

Output:

"HDFC"

9. Constructor / Customizer Before Instance Members

Important: __init__ prepares every object

Before we call instance level members like user3_account.balance, the object needs those members to be created. The constructor, written as __init__, runs automatically when the object is created. You can think of it as the object customizer because it sets the starting data for that object.

class NewBankAccount:
    bank_name: str = "HDFC"

    def __init__(self):
        self.balance = 0
        self.account_number = ""
        self.customer_name = ""

11. Constructor With Custom Values

The constructor can also accept values from outside. This is how each object gets different data.

class NewBankAccount:
    bank_name: str = "HDFC"

    def __init__(self, balance: float, account_number: str, customer_name: str):
        self.balance = balance
        self.account_number = account_number
        self.customer_name = customer_name

    def withdraw(self, amount: float) -> float:
        self.balance -= amount
        return self.balance

13. What self Means

self means the current object. When we call a method using an object, Python passes that object into the method as self.

user4.withdraw(2000)

# Python understands it like:
NewBankAccount.withdraw(user4, 2000)

So inside withdraw, self.balance means user4.balance.

15. Bank Example: main.py

The bank example applies the same OOP concepts in a small working program. It creates two bank account objects and transfers money from RAM to SHYAM.

class BankAccount:
    def __init__(self, balance: float, number: str, name: str):
        self.balance = balance
        self.number = number
        self.name = name

    def withdraw(self, amount: float):
        self.balance -= amount

    def deposit(self, amount: float):
        self.balance += amount

    def transfer(self, account, amount: float):
        self.withdraw(amount)
        account.deposit(amount)

16. BankAccount Structure in main.py

Instance Members

self.balance: account balance

self.number: account number

self.name: account holder name

Instance Methods

withdraw(amount): subtract money

deposit(amount): add money

transfer(account, amount): move money to another account

20. Production Improvement Idea

Validation should be added in real banking code

The current code is excellent for understanding OOP basics. For production code, withdrawals should reject invalid amounts and insufficient balance.

def withdraw(self, amount: float):
    if amount <= 0:
        raise ValueError("Amount must be positive")

    if amount > self.balance:
        raise ValueError("Insufficient balance")

    self.balance -= amount
